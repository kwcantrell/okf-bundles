# Conventions

Agents discover context by convention: where files sit in the tree, what they
are named, and which file is the predictable entry point. These concepts cover
laying out a repository so the nearest instructions win, naming files so their
scope is obvious, and treating `AGENTS.md` as the README agents read.

## Concepts

- [directory layout](/oks/ai-agent-repo-structure/conventions/directory-layout.md) — where context files, skills, and rules live in the tree, including monorepos.
- [naming for discoverability](/oks/ai-agent-repo-structure/conventions/naming-for-discoverability.md) — how file paths and names become the primary discovery signal for agents.
- [README for agents](/oks/ai-agent-repo-structure/conventions/readme-for-agents.md) — `AGENTS.md` as a dedicated, predictable place for agent-facing context.
