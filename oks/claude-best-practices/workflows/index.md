# Workflows

Beyond individual context and planning techniques, Claude Code works best
inside a few repeatable loops that turn "looks done" into a verifiable result.
These concepts cover the documented end-to-end workflows.

## Concepts

- [explore, plan, implement, commit](/oks/claude-best-practices/workflows/explore-plan-code-commit.md) — the four-phase loop for non-trivial changes: research and understand, plan, implement against the plan, then commit.
- [test-driven development](/oks/claude-best-practices/workflows/test-driven-development.md) — write or reproduce a failing test before implementing, then run and iterate until it passes.
- [iterating on visual targets](/oks/claude-best-practices/workflows/iterating-on-visual-targets.md) — give Claude a screenshot or mock as the target, then screenshot its own result and compare to iterate.
