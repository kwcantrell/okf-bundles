# Collaboration Practices

Version control becomes teamwork through a shared review process. On GitHub that
process centers on the pull request: a place to propose changes, discuss them,
and gate them behind review and automated checks before they merge.

## Concepts

- [pull requests](/oks/git-best-practices/collaboration/pull-requests.md) — proposals to merge changes, with conversation, commits, checks, and diff views.
- [code review](/oks/git-best-practices/collaboration/code-review.md) — commenting on, suggesting changes to, and approving or rejecting a pull request.
- [draft pull requests](/oks/git-best-practices/collaboration/draft-prs.md) — work-in-progress PRs that cannot be merged and don't request code-owner review.
- [CODEOWNERS](/oks/git-best-practices/collaboration/codeowners.md) — automatically requesting the right reviewers based on which files change.
