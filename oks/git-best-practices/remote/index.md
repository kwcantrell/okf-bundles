# Working with Remotes

Git is distributed: most operations are local, but collaboration happens by
exchanging commits with remote repositories. These concepts cover how data moves
between your clone and a remote, how local branches relate to remote ones, and
how tags mark and publish released points in history.

## Concepts

- [fetch, pull, push](/oks/git-best-practices/remote/fetch-pull-push.md) — download, integrate, and upload commits between repositories.
- [tracking branches](/oks/git-best-practices/remote/tracking-branches.md) — remote-tracking and upstream relationships that make pull/push automatic.
- [tags and releases](/oks/git-best-practices/remote/tags-and-releases.md) — lightweight vs. annotated tags, pushing tags, and GitHub Releases.
