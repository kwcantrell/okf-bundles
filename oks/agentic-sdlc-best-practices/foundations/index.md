# Foundations

Before any specific practice makes sense, you need the model of *what changed*.
As AI agents absorb the work of writing syntax, the engineer's job relocates
toward expressing intent and verifying outcomes — and that one shift ripples into
a spectrum of how much structure to impose, a new cost model, new roles, and a new
bottleneck. These concepts lay that groundwork.

## Concepts

- [from syntax to intent](/oks/agentic-sdlc-best-practices/foundations/from-syntax-to-intent.md) — the job shift from writing syntax to expressing intent and verifying outcomes, and what the engineer still owns.
- [vibe coding and the spectrum](/oks/agentic-sdlc-best-practices/foundations/vibe-coding-and-the-spectrum.md) — vibe coding vs agentic engineering as endpoints on a spectrum defined by structure, verification, and judgment.
- [when vibe coding is appropriate](/oks/agentic-sdlc-best-practices/foundations/when-vibe-coding-is-appropriate.md) — prototypes and throwaway code vs production, and the crossover point where vibe coding gets more expensive per feature.
- [trust calibration and the 80 percent problem](/oks/agentic-sdlc-best-practices/foundations/trust-calibration-and-the-80-percent-problem.md) — verification as the new bottleneck, the hard last 20%, and calibrating review depth to the stakes.
- [conductor and orchestrator roles](/oks/agentic-sdlc-best-practices/foundations/conductor-and-orchestrator-roles.md) — the two modes developers move between: hands-on real-time direction vs async multi-agent delegation.
- [economics of agentic development](/oks/agentic-sdlc-best-practices/foundations/economics-of-agentic-development.md) — vibe coding's low CapEx / high OpEx hidden debt vs the reverse, with context engineering as the financial lever.
- [anatomy of a coding agent](/oks/agentic-sdlc-best-practices/foundations/anatomy-of-a-coding-agent.md) — model, tools, context, and loop — a refresher on what an agent is, as groundwork for harness engineering.
